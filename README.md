# Rule-Mechanism-AI Three-layer Decoupled Architecture
## Replication Code for "Design and Verification of a Rule-Mechanism-AI 
## Three-layer Collaborative Software Architecture for CZ Crystal Growth"

### Repository Structure

| File | Description |
|---|---|
| simulation_repair3.py | Experiment System B: baseline comparison (Kalman/ARIMA/LSTM) |
| simulation_ablation_repair.py | Experiment System A: ablation study (Strategy A-D) |
| p74_offline_comparison.py | Device P74 offline replay analysis |
| benchmark_latency.py | Prototype system latency benchmark |
| secom_experiment_v4.py | SECOM counterfactual experiment (supplementary) |

### Hardware Requirements
- CPU: Any x86-64 processor
- RAM: 8GB minimum
- GPU: Not required (CPU-only inference)
- OS: Windows 10 / Ubuntu 20.04+

### Installation
pip install -r requirements.txt

### Reproduction
# Experiment System A (Ablation)
python simulation_ablation_repair.py

# Experiment System B (Baseline Comparison)
python simulation_repair3.py

# Device P74 Offline Replay
python p74_offline_comparison.py

### Random Seeds
All experiments use fixed seeds {0, 1, 7, 42, 123} 
for reproducibility. Results may vary slightly 
across Python/library versions.

### Data Availability
- Simulation data: generated by scripts (no external data needed)
- SECOM dataset: UCI ML Repository 
  https://archive.ics.uci.edu/ml/datasets/SECOM
- Device P74 FDC data: proprietary industrial data, 
  not publicly available due to confidentiality agreement
